# Architecture

`neildurrant.com` is the public hub of a small ecosystem. This repo holds only the public, low-sensitivity surfaces; the reading bots live elsewhere for a reason (see [Public/private boundary](#publicprivate-boundary)).

## The ecosystem

| Surface | URL | Lives in | Hosting |
|---|---|---|---|
| Home page | `neildurrant.com` | **this repo** | GitHub Pages |
| Mantrabytes (decoder) | `neildurrant.com/decoder/` | **this repo** | GitHub Pages |
| Tantrabytes | `tantrabytes.neildurrant.com` | `drneildurrant/sanskrit-tantra` (private) | Azure Static Web Apps |
| Monobytes | `monobytes.neildurrant.com` | `drneildurrant/sanskrit-tantra` (private) | Azure Static Web Apps |
| SHiP (unified) | `ship.neildurrant.com` | `drneildurrant/sanskrit-tantra` (private) | Azure Static Web Apps |
| Visualisations | `viz.neildurrant.com` | `drneildurrant/sanskrit-tantra` (private) | Azure Static Web Apps |
| Substack | `substack.neildurrant.com` | — | External (Substack) |

The reading bots also have a **backend** (a retrieval/chat API on Azure Container Apps) that lives in the `sanskrit-tantra` repo; the home page never talks to it.

## This repo is static-only

- `index.html` and `decoder/index.html` are each a single self-contained file — inline `<style>` and `<script>`, no bundler, no framework, no dependencies to install.
- The only runtime data are two same-origin JSON files the home page fetches on load:
  - **`posts.json`** — the "Latest writing" list. Auto-generated from the Substack RSS feed (see [DEPLOY.md](DEPLOY.md#latest-writing-postsjson)). If the fetch fails or JS is off, a static fallback link shows.
  - **`quotes.json`** — pull quotes rotated in the hero. Hand-curated, verbatim from the corpus translations.
- Fonts load from Google Fonts (Cormorant Garamond + Lexend). Everything else is inline.

## Data flows

```
Substack "Provocations" feed ──(weekly GH Action)──▶ scripts/fetch_substack.py ──▶ posts.json ──(fetch)──▶ home page "Latest writing"
quotes.json (hand-edited) ─────────────────────────────────────────────────────────(fetch)──▶ home page rotating pull quote
```

## Public/private boundary

This is the load-bearing architectural decision, so don't "consolidate" it away:

- **This repo is PUBLIC.** It contains only the marketing home page and the decoder — nothing sensitive.
- **`sanskrit-tantra` is PRIVATE.** The bots there bundle ~135 MB of generated reading editions (`web/reader/`, 227 files) that the bots' password gate exists to protect, and which are regenerated by the text-ingest pipeline.

Moving the bot frontends into this public repo would expose that gated content and bloat a Pages repo — so the bots stay private and deploy from their own repo. The two repos share only the **brand system** ([BRAND.md](BRAND.md)), copied into each, not a shared dependency.

## The SHiP unified portal (Phase 1 live, 2026-06-20)

`ship.neildurrant.com` folds **Tantrabytes + Monobytes** behind **one** selector-driven page
(`ship.html`) over the shared master vector index (corpus-tagged rows) and the existing
per-corpus backend. The corpus toggle swaps the whole profile (dropdown, retrieval filter,
reader manifest, examples). A Nietzsche / European-philosophy corpus is the planned third tab.

It stays on the **private** `sanskrit-tantra` side of the [public/private boundary](#publicprivate-boundary)
— same as the bots — because it serves the same gated reading editions; the home page links out
to it but never embeds it. Tantrabytes and Monobytes remain live as their own surfaces for now;
they're slated to retire behind SHiP once the consolidated backend lands.

**Still to come (Phase 2+):** one unpinned `ship-api` (today the page reuses the monobytes API),
retiring the legacy bot pages, and ingesting Nietzsche. **Plan of record:**
`sanskrit-tantra/ROADMAP.md` → "SHiP unified platform — one UI, three corpora".

## Conventions

- **No build step.** Keep pages self-contained; prefer inline CSS/JS over adding tooling.
- **No frameworks / npm.** If a change seems to need a bundler, reconsider — it almost certainly doesn't.
- **Single `main` branch**, deploy-on-push. Test locally before pushing because push = live.
